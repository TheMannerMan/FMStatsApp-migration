# Project Guidelines

## Language

All code must be written in English. This includes variable names, function names, class names, comments, documentation, commit messages, and any other text in the codebase.

The user may communicate in Swedish. Respond in the same language the user writes in, but always produce English code.

## Spec Files

Active specs live in `_specs/`. Implemented specs are archived in `_specs/archive/` — do not read files in `_specs/archive/`. The code is the source of truth for implemented features.

## Plan Files

Active plans live in `_plans/`. Implemented plans are archived in `_plans/archive/` — do not read files in `_plans/archive/`. The code is the source of truth for implemented work.
